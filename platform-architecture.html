Key Architectural Principles

  • Clean Architecture

    Clear separation of concerns with domain-centric approach

  • API-First Design

    Well-defined APIs between all layers

  • Stateless Services

    Services maintain no session state where possible

  • Configuration-Driven

    Behavior configured externally when appropriate

  • Event-Driven Where Beneficial

    Use events for asynchronous processes

Technology Mapping

Backend Technologies

  • API Framework: FastAPI (Python)
  • Authentication: Firebase Authentication
  • Database: PostgreSQL on Google Cloud SQL
  • Transformation: dbt for data modeling
  • Task Processing: Background workers for async processing

Frontend Technologies

  • Framework: Svelte/SvelteKit
  • Visualization: D3.js with Layercake
  • Styling: Tailwind CSS for consistent UI
  • State Management: Svelte stores (no need for external state libraries)

Architecture Summary

This architecture provides a solid foundation while allowing for iterative development and validation of product-market fit. It balances immediate needs with future scalability, ensuring that early development efforts contribute to the long-term vision.
