Introduction

The Marketing Analytics Platform API provides programmatic access to your marketing data, analytics, and insights. This RESTful API allows you to:

  • Manage data source connections (Shopify, Google Analytics, Klaviyo)
  • Access marketing performance metrics and analysis
  • Retrieve actionable recommendations to improve your marketing
  • Configure dashboards and alerts

Base URL

https://api.marketingplatform.example/v1

Response Format

All responses are returned in JSON format with HTTP status codes.

{
  "data": {
    // Response data here
  },
  "meta": {
    // Metadata, pagination info, etc.
  }
}

Authentication

All API requests require authentication using a JSON Web Token (JWT).

Bearer Authentication

Include your JWT token in the Authorization header:

Authorization: Bearer <your_jwt_token>

Obtaining a Token

Authentication is handled through Firebase Authentication. Refer to the Firebase documentation for details on how to authenticate and retrieve a JWT token.

Users

Endpoints for managing user profiles and preferences.

GET /users/me

Get Current User Profile

Returns the profile of the currently authenticated user.

Response: 
{
  "id": "user123",
  "email": "user@example.com",
  "name": "Jane Marketer",
  "role": "owner",
  "created_at": "2025-01-15T10:30:00Z",
  "organization_id": "org456",
  "last_login": "2025-03-28T09:15:22Z"
}
PATCH /users/me

Update Current User Profile

Updates attributes of the currently authenticated user.

Request Body: 
{
  "name": "Jane Smith" // Updated name
}

Data Sources

Endpoints for managing connections to marketing data sources.

GET /data-sources

List All Data Sources

Returns a list of all data sources configured for the organization.

Query Parameters:
Parameter Description
status Filter by connection status (active, pending, error)
type Filter by data source type (shopify, google_analytics, klaviyo)
POST /data-sources

Create a New Data Source

Configure a new data source for the organization.

GET /data-sources/{dataSourceId}

Get Data Source Details

Returns details of a specific data source.

PATCH /data-sources/{dataSourceId}

Update Data Source

Update configuration of a specific data source.

DELETE /data-sources/{dataSourceId}

Delete Data Source

Remove a data source from the organization.

POST /data-sources/{dataSourceId}/sync

Trigger Data Synchronization

Manually trigger a sync from the data source.

Analytics

Endpoints for retrieving marketing analytics and insights.

GET /analytics/metrics

Get Marketing Metrics

Returns marketing performance metrics based on specified parameters.

Required Parameters:
Parameter Description
metrics Comma-separated list of metrics to retrieve
start_date Start date for the analysis period (YYYY-MM-DD)
end_date End date for the analysis period (YYYY-MM-DD)
GET /analytics/funnels

Analyze Conversion Funnel

Returns conversion funnel analysis for the specified steps.

GET /analytics/cohorts

Perform Cohort Analysis

Returns cohort analysis for customer retention and behavior.

GET /analytics/attribution

Get Channel Attribution

Returns attribution analysis for marketing channels.

GET /analytics/health

Get Data Health Metrics

Returns metrics about data completeness, freshness, and quality.

Recommendations

Endpoints for retrieving and managing actionable marketing recommendations.

GET /recommendations

Get Recommendations

Returns actionable marketing recommendations.

Query Parameters:
category Category of recommendations to retrieve
status Filter by recommendation status
GET /recommendations/{recommendationId}

Get Recommendation Details

Returns detailed information about a specific recommendation.

PATCH /recommendations/{recommendationId}

Update Recommendation Status

Update the status of a recommendation (e.g., mark as implemented).

Configuration

GET /config/dashboard

PUT /config/dashboard

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of requests.

HTTP Status Codes

Code Description
200 Success
201 Resource created
204 Success with no content
400 Bad request - invalid parameters
401 Unauthorized - authentication required
403 Forbidden - insufficient permissions
404 Not found - resource doesn't exist
429 Too many requests - rate limit exceeded
500 Server error

Error Response Format

Error responses include a consistent structure with error details:

{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {
      // Additional error details when available
    }
  }
}

Common Error Codes

Code Description
UNAUTHORIZED Authentication is required or has failed
FORBIDDEN User does not have permission for the requested operation
RESOURCE_NOT_FOUND The requested resource could not be found
INVALID_REQUEST The request contains invalid parameters
RATE_LIMIT_EXCEEDED The API rate limit has been exceeded
INTERNAL_ERROR An internal server error occurred

Error Handling Examples

Invalid Parameter

{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Invalid date range specified",
    "details": {
      "start_date": "Must be before end_date",
      "end_date": "Must be after start_date"
    }
  }
}

Resource Not Found

{
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "Data source not found",
    "details": {
      "id": "The specified data source does not exist"
    }
  }
}

Rate Limits

To ensure the stability and availability of the API for all users, rate limits are enforced on API requests.

Default Limits

Plan Rate Limit Burst Limit
Standard 60 requests per minute 100 requests
Premium 120 requests per minute 200 requests
Enterprise Custom Custom

Rate Limit Headers

Rate limit information is included in the response headers:

Header Description
X-RateLimit-Limit The maximum number of requests allowed per minute
X-RateLimit-Remaining The number of requests remaining in the current window
X-RateLimit-Reset The time when the current rate limit window resets (Unix timestamp)

When you exceed the rate limit, you will receive a 429 Too Many Requests response. Please implement exponential backoff or similar strategies to handle rate limiting gracefully.

Pagination

Endpoints that return collections of items support pagination to allow efficient retrieval of large datasets.

Pagination Parameters

Parameter Description Default
page The page number to retrieve 1
per_page The number of items per page 20

Pagination Response

Paginated responses include metadata in the meta section:

{
  "data": [
    // Items for the current page
  ],
  "meta": {
    "current_page": 2,
    "per_page": 20,
    "total_items": 135,
    "total_pages": 7
  }
}

Example: Paginated Request

GET /data-sources?page=2&per_page=15

This will return the second page of data sources with 15 items per page.

The maximum allowed value for per_page is 100. Requesting more than 100 items per page will result in a 400 Bad Request response.
